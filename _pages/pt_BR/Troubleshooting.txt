## <a name="rm_nnid" />Removendo uma NNID sem formatar o 3DS

#### O que você precisa

* A versão mais recente do [GodMode9](https://github.com/d0k3/GodMode9/releases/latest)

#### Instruções

1. Copie o `GodMode9.bin` do `.zip` do GodMode9 para a pasta `/luma/payloads/` no seu cartão SD e o renomeie para `up_GodMode9.bin`
2. Ligue o 3DS segurando o (D-Pad para cima) para entrar no GodMode9 pelo arm9loaderhax
3. Navegue até `SYSNAND CTRNAND` -> `data` -> (ID de 32 caracteres) -> `sysdata` -> `00010038`
4. Segure o gatilho (R), então aperte o X no `00000000` para renomear esse arquivo
5. Aperte o D-Pad para cima uma vez para mudar o nome para `10000000`
6. Aperte (A) para salvar as mudanças
7. Aperte (A) para destravar as modificações na SysNAND, então aperte a sequência de botões pedida
8. Volte para o menu inicial
9. Aperte Start pra reiniciar

## <a name="gw_fbi" />Não consigo injetar o H&S em um N3DS/3DS com downgrade feito pelo Gateway
Isso é causado pela implementação inferior do método de downgrade to Gateway, que deixa duas versões de cada app no sistema. Uma delas não é utilizada, mas confunde o Decrypt9, de modo que o H&S é injetado na versão errada.

#### O que você precisa

* A versão mais recente do [GodMode9](https://github.com/d0k3/GodMode9/releases/latest)

#### Instruções

1. Copie o `GodMode9.bin` do `.zip` do GodMode9 para a pasta `/luma/payloads/` no seu cartão SD e o renomeie para `up_GodMode9.bin`
2. Insira novamente o seu cartão SD no seu 3DS
3. Abra o GodMode9 do arm9loaderhax segurando (Cima) durante a inicialização     
4. Navegue para `SYSNAND CTRNAND` -> `title` -> `00040010`
5. Navegue para a pasta correspondente ao seu dispositivo e região:
  + **Old 3DS EUR**: `00022300` -> `content`
  + **Old 3DS JPN**: `00020300` -> `content`
  + **Old 3DS USA**: `00021300` -> `content`
  + **New 3DS EUR**: `20022300` -> `content`
  + **New 3DS JPN**: `20020300` -> `content`
  + **New 3DS USA**: `20021300` -> `content`
6. Observe que há dois conjuntos de arquivos app e tmd, um com extensão com letras maiúsculas (`.TMD`e `.APP`), and um com extensões com letras minúsculas (`.tmd` e `.app`)
7. Enquanto segura o trigger (R), pressione (Y) para criar um novo diretório
8. Pressione (A) para confirmar o nome `novodiretorio` (não importa como é nomeado)
9. Aperte (A) para destravar as modificações na SysNAND, então aperte a sequência de botões pedida
10. Aperte o trigger (L) em cada um dos arquivos de extensão em maiúsculo (`.TMD`e `.APP`) para marcá-los
11. Pressione (Y) para copiar os arquivos
12. Navegue para o `novodiretorio` ( Nome do diretório nomeado no passo 10)
13. Aperte (Y) para colar os arquivos
14. Selecione "Move path(s)"
15. Os arquivos de extensão em maiúsculo terão assim sido movidos para o `novodiretorio` diretório
16. Aperte (Start) para reiniciar
17. Retorne para [Installing arm9loaderhax](installing-arm9loaderhax) e tente novamente a injeção do FBI
18. Se ainda assim não funcionar, mova os arquivos de extensão em maiúsculo de volta para a pasta `content`, então mova os arquivos de extensão em minúsculo para a pasta `novodiretorio`, retorne para [Installing arm9loaderhax](installing-arm9loaderhax) e tente novamente a injeção do FBI

## <a name="ts_browser" />Um exploit baseado no navegador não está funcionando
Exploits baseados em navegador (como browserhax ou 2xrsa) podem ser instáveis e quebrar frequentemente, mas eles podem em alguns casos serem consertados seguindo os seguintes passos

1. Abra o navegador e, em seguida, abra as configurações do navegador
2. Role até o final e Initialize Savedata (pode ser chamado também de Clear All Save Data)
3. Tente o exploit novamente

3. Copie o `steelhax_payload.bin` para o cartão SD **DO 3DS ALVO**
    + Sobrescreva qualquer arquivos existentes
4. Segure (B) enquanto abre **Steel Diver: Sub Wars**

## <a name="ts_dsiware" />Após realizar o downgrade do DSiWare, meu DSiWare hackeado não funciona

1. Reinicie **o 3DS fonte** enquanto segura o Start para abrir o Hourglass9
2. Vá para SysNAND Backup/Restore e restaure a SysNAND de `NANDmin.bin` (a que você fez antes da transferência do sistema)
3. If you don't see the game at all on **the target 3DS**, link the NNID you bought the game with to **the target 3DS** and redownload it
  + You may have to delete the game from "Data Management" in the "System Settings" first
  + If **the target 3DS** is not the latest version, you may have to run ctr-httpwn to access the eShop
4. On **the source 3DS**, do the the save (and `.app` if you are using the `.app` page) injection steps
5. On **the source 3DS**, go to System Settings, "Data Management", "DSiWare", then copy your DSiWare game to your SD card
6. Either put **the source 3DS**'s SD card in **the target 3DS**, or rename the `Nintendo 3DS` on **the target 3DS**'s SD card and copy the `Nintendo 3DS` folder from **the source 3DS**'s SD card to **the target 3DS**'s SD card
7. On **the target 3DS**, go to System Settings, "Data Management", "DSiWare", then copy your DSiWare game to the system
8. Return your SD cards to normal, then continue with the DSiWare Downgrade

## <a name="ts_d9_backup" />Decrypt9 or Hourglass9 won't restore / can't find my NAND backup

1. Make sure you do not have any folder named "Decrypt9" on **the root** of your SD card
3. Try checking your SD card for errors using [H2testw (Windows)](h2testw-(windows)), [F3 (Linux)](f3-(linux)), or [F3X (Mac)](f3x-(mac))
4. Try backing up your SD card files, then formatting it and copying them back
5. Try a different SD card

## <a name="ts_sys_down" />Black screen on SysNAND boot

1. Try booting with your SD card out, and then reinserting it after booting.
    1. Power off your 3DS by holding down the power button.
    2. Take out the SD card.
    3. Power on the 3DS.
    4. When the home menu appears, reinsert your SD card.
    5. If this worked, you should Clear Home Menu's extdata by navigating to the following folder on your SD card: `/Nintendo 3DS/(32 Character ID)/(32 Character ID)/extdata/00000000/`
        + Região EUR: Delete `00000098`
        + Região JPN: Delete `00000082`
        + Região USA: Delete `0000008f`
        + Região KOR: Delete `000000A9`
2. Try booting without any cartridges inserted (including flashcarts)
3. If you have a hardmod and a NAND backup, flash the backup back to SysNAND.
4. Try booting into recovery mode and updating your system.    
    *This probably will not work for an Old 3DS downgraded to 2.1.0*    
    **This will BRICK a New 3DS downgraded to 2.1.0**
    1. Power off your 3DS by holding down the power button.
    2. Hold L+R+A+Up.
    3. Power on the 3DS.
    4. If you enter safe mode, update your 3DS *only if you have an entrypoint for the latest FW version and it is possible to downgrade from it* and attempt the downgrade again.
5. Your 3DS may be bricked. For support, ask for help at [#3dshacks on Rizon IRC](https://www.reddit.com/r/3dshacks/wiki/irc) or [3DS Hacking on Discord](https://discord.gg/MWxPgEp).

## <a name="ts_sys_a9lh" />Black screen on SysNAND boot after installing arm9loaderhax

1. Ensure you have a working payload.
    1. Check for the existence of `arm9loaderhax.bin` in the root of your SD card.
2. Try resetting Luma's config and fix options
    1. Delete `/luma/config.bin` from your SD card
    2. Sete suas opções quando ele bootar
3. Tente bootar o Hourglass9
    1. On Luma3DS, hold (Start) on boot
4. Try deleting home menu's extdata
    1. Clear Home Menu's extdata by navigating to the following folder on your SD card: `/Nintendo 3DS/(32 Character ID)/(32 Character ID)/extdata/00000000/`
        + Região EUR: Delete `00000098`
        + Região JPN: Delete `00000082`
        + Região USA: Delete `0000008f`
        + Região KOR: Delete `000000A9`
5. Try booting without any cartridges inserted (including flashcarts)
6. If you previously downgraded with Gateway, ensure that you are using the latest Luma3DS version (v6.2.3 or higher, at the least)
7. If your NAND is of a version between 3.0.0 and 4.5.0, do the following:
    + Ensure that you are using the latest Luma3DS version (v6.6 or higher, at the least)
