Ein gültiges NAND Backup sollte in den meisten fällen eine der folgenden Größen besitzen.
{: .notice--primary}

Gelegentlich kann die Größe des Backups von der hier gelisteten abweichen. Es ist in Ordnung wenn das Backup größer als Gedacht ist, vergewissere dich lediglich, dass sie niemals *kleiner* als die hier gelisteten Größen ist.
{: .notice--info}

Beachte das ein NAND Backup mit der *Mindestgröße* immer die kleinstmögliche Größe aufweist um Platz zu sparen und eventuell nicht die gleiche Größe hat wie ein anderes Backup auf deiner Konsole. Das ist in Ordnung, vergewissere dich jedoch, dass es einer der hier gelisteten Größen entspricht.
{: .notice--info}

Der Name bezieht sich auf den Hersteller des NAND-Chips im Gerät, *nicht* der SD-Karte.
{: .notice--warning}

**Dein Backup sollte einer der Größen auf dieser Seite entsprechen; Wenn dies nicht der Fall ist, lösche es und erstelle ein neues! **
{: .notice--danger}

## 2DS:    

    + Toshiba NAND:     
      988.807.168 bytes = 943 MiB    

    + Toshiba NAND:    
      1.979.711.488 bytes = 1.84 GiB    

    + Samsung NAND:    
      1.000.341.504 bytes = 954 MiB    

    + Samsung NAND:    
      1.300.234.240 bytes = 1.21 GiB    

## Old 3DS:    

    + Toshiba NAND:     
      988.807.168 bytes = 943 MiB    

    + Samsung NAND:    
      1.000.341.504 bytes = 954 MiB    

## New 3DS:    

    + Toshiba NAND:    
      1.979.711.488 bytes = 1.84 GiB    

    + Samsung NAND:    
      1.300.234.240 bytes = 1.21 GiB    

    + Samsung NAND:
      1.954.545.664 bytes = 1.82 GiB
