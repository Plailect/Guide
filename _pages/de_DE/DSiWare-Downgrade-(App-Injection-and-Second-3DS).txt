Wenn du zwischen den Versionen 11.0.0 und 11.2.0 bist, kannst du diesem Guide folgen, um deine NATIVE_FIRM mithilfe einer DSIWare und eines zweiten 3DS mit bereits installierter Custom Firmware downzugraden oder dein NAND zu sichern, bzw. wiederherzustellen.
{: .notice}

Wenn du unter 11.2.0 auf einem der beiden Geräte bist, solltest du die ctr-httpwn - Schritte (wenn du dazu aufgefordert wirst) auf jedem Gerät unter 11.2.0 ausführen, damit ein System-Transfer zwischen den Geräten möglich ist.
{: .notice--info}

Dies nutzt eine Schwachstelle, welche es DSIWare-Titeln erlaubt, überall im NAND zu lesen und zu schreiben.
{: .notice--info}

Dies ist eine momentan funktionierende Implementierung des "FIRM partitions known-plaintext"-Exploits, der [hier](https://www.3dbrew.org/wiki/3DS_System_Flaws) erklärt wird.
{: .notice--info}

Dieser Guide geht davon aus, dass der CFW 3DS bereits mit arm9loaderhax läuft und mit diesem Guide gehackt wurde.
Trotzdem sollte er auch (mit kleinen Modifikationen, wie z. B. die Durchführung aller SysNAND-Schritte auf dem EmuNAND) auf EmuNAND-Systemen funktionieren. Beachte, dass die Begriffe EmuNAND und RedNAND sich auf leicht unterschiedliche Implemenhierungen [des selben Konzepts](http://3dbrew.org/wiki/NAND_Redirection) beziehen.
{: .notice--info}

Deine DSIWare-Spielstände werden gesichert, bevor sie mit gehackten Spielständen ersetzt werden.
{: .notice--info}

{% capture notice-4 %}
Dieser Exploit erfordert einen[System Transfer](http://en-americas-support.nintendo.com/app/answers/detail/a_id/13996/) von einem CFW 3DS zu einem regulären 3DS als Teil der benötigten Schritte. System Transfers funktionieren *nur* in die folgenden Richtungen:    
  + New 3DS -> New 3DS    
  + Old 3DS oder 2DS -> Old 3DS oder 2DS    
  + Old 3DS oder 2DS -> New 3DS    
{% endcapture %}

<div class="notice--warning">{{ notice-4 | markdownify }}</div>

Beide Systeme MÜSSEN aus der selben Region sein.
{: .notice--warning}

**Alle Daten des Ziel-3DS werden gelöscht!**
{: .notice--danger}

Die NNID des CFW 3DS wird an den Ziel-3DS gebunden sein, bis du entweder einen System Transfer zurück durchführst (nach einer Woche Wartezeit) oder dich an den Nintendo Support wendest, um die Bindung aufzuheben! (Details weiter unten)
{: .notice--danger}

System Transfers können nur einmal pro Woche durchgeführt werden.
{: .notice--danger}

#### Was du brauchst

* Zwei 3DS Systeme
  + **Der Quell-3DS**: Der 3DS mit einer installierten Custom Firmware (arm9loaderhax oder eine Variante von EmuNAND/RedNAND) *Auf der aktuellen Version*
  + **Der Ziel-3DS:**: Der ungehackte 3DS mit regulärer Firmware*(zwischen 11.0.0 und 11.2.0)*
* Kaufe (falls noch nicht getan) und installiere ein kompatibles DsiWare-Spiel aus dem eShop auf dem **Quell-3DS**
  + Eine illegale Kopie des Spiels wird **Nicht** funktionieren
  + Eine Liste mit kompatiblen Spielen findest du unter [DSiWare List](dsiware-list)
* Ein Einstiegspunkt aus [Homebrew Launcher (SoundHax)](homebrew-launcher-(soundhax)) oder [Homebrew Launcher (Kein Browser)](homebrew-launcher-(no-browser))

#### Anleitung

##### Abschnitt I - Vorbereitungen

Benutze einen [save manager](https://github.com/J-D-K/JKSM/releases/latest) um alle Spielstände auf dem Ziel-3DS, die dir wichtig sind, zu sichern (Er wird formatiert!)
{: .notice--warning}

1. Erstelle einen Ordner namens `files9` im Stammverzeichnis der SD-Karte des **Quell-3DS**, falls er noch nicht existiert
2. Kopiere `GodMode9.bin` aus der GodMode9 `.zip` in den `/luma/payloads/` Ordner auf der **Quell-3DS** SD-Karte und benenne `GodMode9.bin` in `/luma/payloads/` um zu `up_GodMode9.bin`
3. Kopiere die `.app` aus der DSiWare injection `.zip` ins Stammverzeichnis der **Quell-3DS** SD-Karte
4. Kopiere den `savedata`-Ordner aus der DSiWare injection `.zip` ins Stammverzeichnis der **Quell-3DS** SD-Karte
5. Setze die **Quell-3DS** SD-Karte wieder in das entsprechende System ein
6. On **the source 3DS**, hold Start on boot to launch Hourglass9
7. Go to SysNAND Options, then SysNAND Backup/Restore, then backup **(min size)** SysNAND to `NANDmin.bin`
8. Press (Select) on the main menu to eject **the source 3DS**'s SD card, then put it in your computer

##### Section V - System Transfer

1. **Backup every file on both 3DS's SD cards to two separate folders on your computer (keep track of which is which)!**
2. Reinsert each SD card back into their corresponding 3DS
3. If **the target 3DS** has a Nintendo Network ID on it, you must format the device using System Settings:
  + Go to the last page of "Other Settings" and select "Format System Memory", then follow all instructions
4. Read the following:
  + Your CFW 3DS = the source 3DS = "Source System"
  + Your Stock 3DS = the target 3DS = "Target System"
  + **Move DSiWare titles if prompted!**
  + Do **NOT** delete the source system's SD card contents if prompted
  + Make sure neither device's battery dies during the transfer
  + 2DS/Old 3DS (source) to New 3DS (target) only - if asked which method you wish to use to transfer the SD card data:
    + **Do NOT** choose the "Low-Capacity microSD Card Transfer" or minimal option (option 2), it will only transfer tickets and likely will not transfer the DSiWare save.
    + Fast Method: If you have the ability to move the data from the SD card (source) to the microSD card (target), when prompted use the "PC-Based Transfer" option (option 3).
    + Slowest Method: If you don't have the ability to move the data on a PC use the **full** "Wireless Transfer" option (option 1).
5. Go to [this link](http://en-americas-support.nintendo.com/app/answers/detail/a_id/227/) and follow Nintendo's official instructions for System Transferring from one system to another while keeping in mind what you just read
