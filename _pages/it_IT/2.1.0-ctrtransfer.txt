Se effettui il downgrade alla versione 2.1.0 su un 2DS o su un New 3DS lasciando inattiva la comunicazione wireless, puoi riattivarla rimuovendo la batteria per alcuni secondi prima di accendere la console.
{: .notice--info}

Se stai usando la Gestione microSD del New 3DS per trasferire i file sulla tua scheda SD, tieni presente che non è utilizzabile nella versione 2.1.0. Assicurati di avere un lettore di microSD a portata di mano prima di proseguire.
{: .notice--info}

La tua console potrebbe non visualizzare più i giochi installati dopo il ctrtransfer. La causa è la rimozione dei ticket durante il trasferimento, per cui è necessario effettuarne il backup. Verranno ripristinati al termine della guida.
{: .notice--info}

**Al momento, la connessione a internet è NECESSARIA per proseguire dopo il ctrtransfer 2.1.0.**
{: .notice--warning}

**Assicurati di disattivare il Filtro famiglia della tua console prima del ctrtransfer 2.1.0. Se non conosci la password, collegati a [questo](http://mkey.salthax.org/) sito.**
{: .notice--warning}

**Un ctrtransfer rimuoverà dalla console tutti i ticket installati dall'utente fintantoché non verrà ripristinato un backup. La creazione di un backup è pertanto NECESSARIA, altrimenti PERDERAI l'accesso ai salvataggi e a tutti i giochi installati!**
{: .notice--danger}

**Non formattare mai un 2DS se sei ad una versione <6.0.0 o non potrai più completare l'installazione iniziale e BRICKERAI la console!**
{: .notice--danger}

**Mai aggiornare un New 3DS se sei alla versione 2.1.0 (firmware di un Old 3DS) o BRICKERAI LA CONSOLE! Prima si deve OBBLIGATORIAMENTE ripristinare un backup della NAND proveniente dal dispositivo o effettuare un ctrtransfer di un firmware standard del New 3DS!**
{: .notice--danger}

#### Panoramica

In questa sezione verrà flashata la versione 2.1.0 nella partizione [CTRNAND](https://www.3dbrew.org/wiki/Flash_Filesystem#CTR_partition) per sfruttare un exploit che permetterà di estrarre l'[OTP](otp-info) univoco della tua console. Questo file OTP è necessario per l'installazione dell'arm9loaderhax, e **non** può essere condiviso con altre console.

Per farlo, verrà [installata un'immagine ctrtransfer prefabbricata](https://www.reddit.com/r/3dshacks/comments/4zhe4a/) contenente la versione 2.1.0, copiando nella tua console dei file specifici (come `moveable.sed` e `SecureInfo_A`) per poi correggere il title database.

Il file OTP è necessario per utilizzare l'arm9loaderhax.

#### Istruzioni

##### Sezione I - Preparazione

Questi passi vanno eseguiti all'interno di Decrypt9.

1. Premi (Select) nel menu principale per rimuovere la scheda SD, poi inseriscila nel tuo computer
2. Copia i file 2.1.0 `.bin` e `.bin.sha` dall'archivio `.zip` di ctrtransfer alla cartella `/files9/` della tua scheda SD
3. Reinserisci la tua scheda SD nel tuo 3DS
4. Premi (B) per tornare al menu principale del Decrypt9

##### Sezione II - ctrtransfer

**E' INDISPENSABILE aver a disposizione un backup della NAND proveniente dal dispositivo. Verrà comunque chiesto di crearne uno durante le operazioni nel caso in cui non fosse già stato creato.**
{: .notice--danger}

**Saltare il backup porterà ad una PERDITA di tutti i giochi installati e dei salvataggi!**
{: .notice--danger}

**Se per qualsiasi motivo ancora non si ha a disposizione il backup della NAND, occorre crearne uno non appena richiesto premendo (A).**
{: .notice--danger}

**Se non si dispone di sufficiente spazio libero per creare il backup è comunque possibile utilizzare una scheda SD differente, oppure occorre liberare spazio dalla scheda SD. Premendo (Select) nel menu di Decrypt9WIP si potrà estrarre la scheda SD e sostituirla con un'altra o procedere alla pulizia tramite PC**
{: .notice--danger}

4. Selezionare l'opzione "SysNAND Options", poi selezionare "CTRNAND Transfer" e infine selezionare "Auto CTRNAND Transfer"
5. Selezionare l'immagine ctrtransfer 2.1.0 quando richiesto tramite la pressione del tasto (A), quindi inserire la combinazione di tasti richiesta a schermo
6. Nel caso non si abbia ancora possesso di un backup della NAND, **effettuare un backup della sysNAND nel file 'NANDmin.bin' quando richiesto premendo il tasto (A)**
  +**Essere in possesso di un backup della NAND proveniente dal dispositivo è NECESSARIO.**
  +**Saltare la creazione del backup risulterà nella PERDITA di tutti i giochi installati e dei salvataggi!**
7. Permettere al processo di trasferimento di completarsi automaticamente, potrebbe richiedere del tempo
8. A trasferimento completato premere (B) per tornare indietro
9. Premere (Select) per estrarre la scheda SD
10. Cancellare i file immagine ctrtransfer 2.1.0 con estensione '.bin' e '.bin.sha' dalla cartella '/files9/' presente sulla scheda SD
11. Premere (Start) per riavviare il dispositivo senza reinserire la scheda SD
  + Con la versione 2.1.0 attiva, il dispositivo mostrerà una schermata nera all'avvio se la scheda SD sarà stata inserita prima che il menu home abbia finito il caricamento
12. Reinserire la scheda SD dentro la console dopo che il menu home ha finito di caricare
  + Ogni volta che il dispositivo è avviato con la versione 2.1.0 è necessario rimuovere la scheda SD prima dell'accensione e inserirla dopo il caricamento del menu home
  + Questo problema sarà risolto con il ripristino del dispositivo nella prossima pagina

___

Il primo avvio in versione 2.1.0 può richiedere più tempo rispetto ad un normale avvio su alcuni dispositvi.
{: .notice--info}

*(Distorsioni nello schermo o scoloriture sono normali per alcuni dispositivi con il sistema 2.1.0, tuttò tornerà alla normalità una volta ripristinato il backup)*
