Questa sezione aggiuntiva mostra come cambiare regione su arm9loaderhax CFW SysNAND. Ciò si ottiene installando l'immagine ctrtransfer 9.2.0 relativa alla regione con cui si vuole cambiare e modificando il file SecureInfo_A.
{: .notice--primary}

**Il cambio regione è una pratica quasi completamente inutile in quanto Luma3DS supporta l'avvio di giochi fuori regione e l'[emulazione regionale](https://github.com/AuroraWright/Luma3DS/wiki/Options-and-usage) individuale.**
{: .notice--info}

Questo processo scollegherà il NNID dal sistema, dato che d'ora in poi non sarà compatibile. I NNID sono legati alla regione del dispositivo che li crea e non possono subire trasferimenti di regione senza un [processo avanzato e molto complesso](https://gist.githubusercontent.com/yifanlu/e80db121d38aceb8cca0e03cefd5853b/raw/3c4dd89869156ca0f945a2791e699acfdb32b510/gistfile1.txt).
{: .notice--warning}

Dopo questo processo, solo i **New 3DS che non hanno mai avuto accesso all'eShop** e **qualsiasi Old 3DS** potranno accedere all'eShop dopo aver creato un nuovo NNID legato alla nuova regione. I New 3DS con regione modificata che hanno già avuto accesso all'eShop sulla regione originale non potranno creare un nuovo NNID e quindi accedere all'eShop nella nuova regione!
{: .notice--warning}

Da notare che occasionalmente l'eShop inspiegabilmente non funzionerà con alcuni dispositivi con cui avrebbe dovuto. Questo è interamente dovuto ai server Nintendo e, su questa cosa, non possiamo fare assolutamente nulla.
{: .notice--warning}

Il cambio regione utilizzando il ctrtransfer sembra far smettere di funzionare le patch che si applicano al riavvio. La causa di ciò è sconosciuta, al momento gli utenti in possesso di un Old 3DS sono impossibilitati ad avviare giochi che utilizzano la modalità di memoria estesa (ad esempio Monster Hunter, Super Smash Bros e Pokémon Sole / Luna).
{: .notice--warning}

**È necessario aver già installato arm9loaderhax e Luma3DS per poter procedere**
{: .notice--danger}

#### Istruzioni

**Alcune console hanno il file `SecureInfo_B` invece di `SecureInfo_A`; non è un problema e basta utilizzare `SecureInfo_B` nei prossimi passi.**    

##### Sezione I - Preparazione

1. Copia `GodMode9.bin` dall'archivio GodMode9 `.zip` alla cartella `/luma/payloads/` presente nella scheda SD e rinomina il file `GodMode9.bin` nella cartella `/luma/payloads/` in`up_GodMode9.bin`
2. Copia il file `Decrypt9WIP.bin` all'interno della cartella `/luma/payloads/` contenuta nella scheda SD e rinomina il file `Decrypt9WIP.bin` in `x_Decrypt9WIP.bin`
3. Copia i file del ctrtransfer 9.2.0 con estensione `.bin` and `.bin.sha` dall'archivio `.zip` ctrtransfer alla cartella `/files9/` presente nella scheda SD
4. Reinserisci la tua scheda SD nel tuo 3DS

##### Sezione II - Backup dei ticket e della configurazione di sistema

5. Premi il pulsante (Start) mentre tieni premuto il pulsante (X) per riavviare il sistema in Decrypt9 tramite arm9loaderhax     
6. Seleziona "Ticket/Titlekey Options", quindi seleziona "Ticket Dump (SysNAND)"
7. Premi (Select) per rimuovere la scheda SD, poi inseriscila nel tuo computer
8. Elimina tutti i file con estensione ".tik" presenti nella cartella "/files9/" che **NON** cominciano con "00-""
9. Lascia tutti i file con estensione ".tik" che cominciano per "00-" nella cartella
10. Reinserisci la scheda SD nel tuo 3DS, quindi premi (B) per tornare al menu principale
11. Seleziona "SysNAND Options", quindi "System Save Dump", ed infine "Dump configsave.bin"

##### Sezione III - ctrtransfer

6. Selezionare l'opzione "SysNAND Options", poi selezionare "CTRNAND Transfer" e infine selezionare "Auto CTRNAND Transfer"
7. Quando richiesto, seleziona il file del ctrtransfer 9.2.0 premendo (A)
8. Se non hai un backup della tua NAND, **fai il backup della SysNAND nel file `NANDmin.bin` quando richiesto premendo (A)**
9. Permetti al processo di trasferimento di completarsi automaticamente, potrebbe richiedere del tempo
10. Appena il trasferimento è completato, premi (Select) per estrarre la scheda SD
11. Inserisci la scheda SD nel computer, quindi copia i file "NANDmin.bin" e "NANDmin.bin.sha" dalla cartella "/files9/" presente nella scheda SD ad una posizione sicura
  + Crea delle copie e mettile in più locazioni
  + Questo backup ti salverà da un brick nel caso succeda qualcosa in futuro
  + **Il file di backup dovrà avere l'esatta dimensione riportata su [questa](nand-size) pagina; se così non fosse dovresti cancellarlo e crearne uno nuovo!**
12. Dopo averlo copiato, cancella i file immagine del ctrtransfer 9.2.0 con estensione `.bin` and `.bin.sha` dalla cartella `/files9/`presente nella scheda SD
13. Reinserisci la tua scheda SD nel tuo 3DS

##### Sezione IV - Modifica delle SecureInfo

1. Premi il pulsante (Start) mentre tieni premuto il pulsante (Su) per riavviare il sistema in GodMode9 tramite harm9loaderhax     
2. Naviga in `SYSNAND CTRNAND` -> `rw` -> `sys`
3. Premi il pulsante (Y) sopra `SecureInfo_A` per copiarlo
4. Premi il pulsante (Y) per incollare una copia di `SecureInfo_A`
5. Seleziona "Copy path(s)"
6. Premi il pulsante (A) per sbloccare la scrittura su SysNAND quindi inserisci la combinazione di tasti richiesta a schermo
7. Seleziona "Choose new name"
8. Usa la croce direzionale per rinominare il file in `SecureInfo_C`, premi il pulsante (A) quando hai finito (sovrascrivi qualsiasi `SecureInfo_C` già esistente)
9. Scorri fino a selezionare il file `SecureInfo_C` che hai appena incollato
10. Premi il pulsante (A) sulla selezione `SecureInfo_C` quindi seleziona "Show in Hexeditor"
11. Premi il pulsante (A) per continuare
12. Premi il pulsante (A) per attivare la modalità modifica
13. Scorri fino all'inizio della riga 00000100 e premi il pulsante (A)
14. Tieni premuto il pulsante (A) mentre prmi i pulsanti (SU) o (GIÚ) della croce direzionale per modificare il numero
15. Cambia la prima coppia di numeri nella linea 00000100 con una delle seguenti in base *alla regione in cui vuoi cambiare*:
    - "00" : JPN
    - "01" : USA
    - "02" : EUR
16. Press (B) to exit edit mode, then save changes, then press (B) again to return to the directory
17. You should see both a `SecureInfo_A` and `SecureInfo_C` if everything was done properly. (`SecureInfo_C` is used in place of `SecureInfo_A` when using Luma3DS if it detects the file on boot)
18. Press Start to reboot

##### Section V - Reinstalling Tickets

1. Open FBI
2. Select "SD"
3. Select "files9"
4. Select "\<current directory>"
5. Select "Install and delete all tickets"
6. Wait. The system may appear to freeze, just give it time.
7. Press (A) to confirm
8. Press (B) to decline installing tickets from CDN.
9. Exit with the home button

##### Section VI - Region settings

1. Open the System Settings
2. Go to "Other Settings", "Profile", then "Region Settings"
3. Select a country from the region you switched to
4. If prompted, you do not need to set a state
5. Update your region changed CFW SysNAND to the latest version

##### Section VII - Restore config save

1. Reboot holding (X) during boot to launch arm9loaderhax Decrypt9
2. Go to "SysNAND Options", then "System Save Inject", then select "Inject configsave.bin"

Old 3DSs / 2DSs may need to format (using TinyFormat or the System Settings) to play extended memory mode games (such as Monster Hunter, Super Smash Bros, or Pokemon Sun/Moon).
{: .notice--info}

Formatting a device following a ctrtransfer *without restoring the config save* can cause screen brightness / flickering issues. See [this](https://github.com/Plailect/Guide/issues/794).
{: .notice--info}
