El paso final de esta guía es instalar arm9loaderhax y configurar Luma3DS para que se ejecute a pocos milisegundos después de encender tu consola. Esto se logra usando SafeA9LHInstaller hecho por [AuroraWright](https://github.com/AuroraWright/).
{: .notice}

Esto instalará [la versión de AuroraWright](https://github.com/AuroraWright/arm9loaderhax) de arm9loaderhax.
{: .notice--info}

También vamos a configurar la posibilidad de ejecutar payloads desde arm9loaderhax. Esto nos brinda la posibilidad de desbrickear nuestra sysNAND restaurando un backup.
{: .notice--info}

**No se puede usar el OTP de otra consola o GARANTIZADO que tu consola se va a brickear**
{: .notice--danger}

#### Descripción general de pasos

En esta sección vamos a realizar el proceso para el que hicimos todos los pasos anteriores: finalmente instalar arm9loaderhax.

Este es uno de los mejores exploits porque se instala permanentemente en la memoria NAND de nuestra consola, se ejecuta antes que cargue el sistema y eso le permite no solamente funcionar en *todas* las versiones una vez instalado sino que también protegerse y recuperarse de situaciones que brickearian una consola normal, como ser un fallo en el menú de la consola o un juego mal instalado.

El archivo `arm9loaderhax.bin` es lo que arm9loaderhax ejecuta cuando termina de cargar y puede ser cualquier archivo de payload para arm9. Este archivo puede ser reemplazado en cualquier momento, sin embargo Luma3DS nos permite encadenar otros payloads si mantenemos apretado algún botón durante el encendido de nuestra consola.

En este caso usamos Luma3ds hecho por [AuroraWright](https://github.com/AuroraWright/) para iniciar a sysNAND parcheada directamente. Esto simplifica considerablemente el uso de una 3DS hackeada y nos deja mucho mas espacio libre en la tarjeta SD comparado con el uso de un sistema de emuNAND.

Una vez instalado arm9loaderhax y configurado Luma3DS, vamos a restaurar el backup que hicimos previamente.

Durante este proceso también instalamos los siguientes programas:    

+  **FBI** *(instala juegos y aplicaciones en archivos .CIA)*
+  **Luma3DS Updater** *(actualizador para nuestro CFW)*
+  **Hourglass9** *(herramienta multiuso que puede hacer respaldos de la NAND y de cartuchos)*

### Instrucciones

##### Sección I - Preparativos

{% capture notice-5 %}
** Asegúrate de estar utilizando una tarjeta SD que no está dañada! **

** Si utilizas una tarjeta SD corrupta sin arreglarla puedes BRICKEAR! **

** Si crees que la tarjeta SD puede estar dañada, compruebe su tarjeta SD usando [H2testw (Windows)](h2testw-(windows)), [F3 (Linux)](f3-(linux)), o [F3X (Mac)](f3x-(mac))! **
{% endcapture %}

<div class="notice--danger">{{ notice-5 | markdownify }}</div>

1. ** Si existe, copia la carpeta 'files9' de tu tarjeta SD a un lugar seguro en tu computadora y crea backups por todos lados (por ejemplo, en servicios de almacenamiento online de archivos). Los archivos de dentro de dicha carpeta podrían servirte para recuperar tus datos o incluso revivir tu consola en caso de que algo salga mal.**
2. Crea una carpeta llamada `cias` en la raíz de tu tarjeta SD si aún no existe
4. **Elimina la carpeta 'a9lh' de la raíz de tu tarjeta SD si existe**
  + ** Si accidentalmente instalas arm9loaderhax usando el 'OTP.bin' de otro dispositivo, vas a BRICKEAR! **
3. Elimina la carpeta '3ds' de la raíz de tu tarjeta SD si existe
4. **Copia _el contenido_ de starter.zip a la raíz de tu tarjeta SD**
  + Esto incluye una nueva carpeta de '3ds' para sustituir a la que eliminaste
5. Copia _el contenido_ de SafeA9LHInstaller '.7z' a la raíz de tu tarjeta SD
6. **Copia la carpeta 'a9lh' dentro de 'data_input_v4.zip' a la raíz de tu tarjeta SD**
7. **Copia _el contenido_ de arm9loaderhax '.7z' a la carpeta 'a9lh' en tu tarjeta SD**
9. Copia `hblauncher_loader.cia` del archivo `.zip` de hblauncher_loader a la carpeta `/cias/` de tu tarjeta SD
10. Copia `lumaupdater.cia` del archivo `.zip` de Luma3DS Updater a la carpeta `/cias/` en tu tarjeta SD
11. Copia `FBI.cia` del archivo `.zip`de FBI a la carpeta `/cias/`en tu tarjeta SD
12. **Copia `arm9loaderhax.bin` desde el archivo `.7z` de Luma3DS a la raíz de tu tarjeta SD, sobrescribe los archivos existentes**
13. Crea una carpeta `luma` en la raíz de tu tarjeta SD
14. Crea una carpeta `payloads` dentro de la carpeta `luma` en tu tarjeta SD
15. Copia 'Hourglass9.bin' del archivo '.zip' de Hourglass9 a la carpeta '/luma/payloads/' en tu tarjeta SD y cambia el nombre de 'Hourglass9.bin' a 'start_Hourglass9.bin'
16. Copia `GodMode9.bin` del archivo `.zip` de GodMode9 a la carpeta `/luma/payloads/` en tu tarjeta SD y cambia el nombre de `GodMode9.bin` a `up_GodMode9.bin`
16. Copia 'aeskeydb.bin' a la carpeta '/files9/' en tu tarjeta SD
17. Copia 'DspDump.3dsx' a la carpeta '/ 3ds /' en tu tarjeta SD
18. Copia _el contenido_ de `fbi-2.4.2-injectable.zip` a la carpeta `/files9/` en tu tarjeta SD

##### Sección II - Instalar arm9loaderhax

1. Re-inserta la tarjeta SD en tu 3DS
2. Sigue los pasos para instalar arm9loaderhax en tu consola:
  + Tenés que estar en 2.1.0
  + Navegá a `http://dukesrg.github.io/2xrsa.html?arm11.bin` desde tu 3DS
    + Si te sale el error "este servicio no está disponible en su región", usá la configuración del sistema para cambiar el país de tu consola para que coincida con la región NAND instalaste cuando hiciste el 2.1.0 ctrtransfer
    + Si obtenés otro error, [seguí esta guía de errores]
(troubleshooting#ts_browser)
    + Si tu pantalla muestra graficos raros o lineas de colores (glitches), [seguí esta guía de errores](troubleshooting#ts_safe_a9lh_screen)
  + Apretá (Select) para instalar
  + El programa de instalación instalará ahora arm9loaderhax en tu consola (esto es muy rápido)
  + Apagá la consola, mantené presionado el botón de encendido hasta que se apague si hace falta
  + Copia el archivo especifico de tu consola `OTP.bin` de la carpeta `/a9lh/` en tu tarjeta SD a un lugar seguro y creá backups por todos lados (por ejemplo, en servicios de almacenamiento online de archivos), despues re-inserta la tarjeta SD en tu 3DS

##### Section III - Configurar Luma3DS

1. Hold select on boot to enter the Luma3DS menu
  + Make sure to start holding the button before pressing power
  + If you get a black screen, [follow this troubleshooting guide](troubleshooting#ts_sys_a9lh)   
  + If you boot to SafeA9LHInstaller, [follow this troubleshooting guide](troubleshooting#ts_safe_a9lh)
2. Use the (A) button and the D-Pad to turn on the following:    
  + **"Autoboot SysNAND"**
  + **"Use SysNAND FIRM if booting with R"**
  + **"Show NAND or user string in System Settings"**
3. If you are using a **New 3DS**, you should *also* enable the following:
  + **"New 3DS CPU" to "Clock+L2(x)"**
    + This will increase the framerate of many games, but may cause instability in others
    + If some games do not work properly, disable this option and try again
4. Press Start to save and reboot
  + If you get a black screen, just continue to the next section   
  + If you get a "Failed to mount CTRNAND" error, just continue to the next section     

##### Section IV - Restoring the System

If, before following this guide, you already had an EmuNAND setup and would like to move the contents of your previous EmuNAND to your new SysNAND CFW, now is the time to [follow Move EmuNAND](move-emunand) instead of doing the first four steps in this section.
{: .notice--info}

1. Open Hourglass9 from arm9loaderhax by holding (Start) on boot
2. Go to "SysNAND Backup/Restore"
3. Go to "SysNAND Restore (keep a9lh)" and restore from `NANDmin.bin`
4. Press (Start) to reboot
  + If you get a black screen, [follow 9.2.0 ctrtransfer](9.2.0-ctrtransfer)
5. If your backup was of a version between 3.0.0 and 4.5.0, your console will not boot until you manually download the required firmware:
  + Download [this file](http://nus.cdn.c.shop.nintendowifi.net/ccs/download/0004013800000002/00000056) and rename it to `firmware.bin`
  + Download [this file](http://nus.cdn.c.shop.nintendowifi.net/ccs/download/0004013800000002/cetk)
  + Copy `firmware.bin` and `cetk` to the `/luma/` folder on your SD card
  + Delete both of these files after updating your 3DS
2. Update your 3DS by going to System Settings, then "Other Settings", then going all the way to the right and using "System Update"
  + Updates while using A9LH + Luma (what you have) are safe, stop asking about it
  + The warning about never updating a New 3DS from 2.1.0 does not apply after restoring your NAND backup.
  + If this gives you an error, set your DNS settings to "auto"
  + If this still gives you an error and your NAND is below 9.2.0, [follow 9.2.0 ctrtransfer](9.2.0-ctrtransfer)


##### Section V - Injecting FBI

1. Open Hourglass9 from arm9loaderhax by holding (Start) on boot
2. Go to "SysNAND Backup/Restore", then select the "Health&Safety Dump" option to dump Health & Safety to `hs.app` **(you can use Up and Down / Left and Right to change the name)**
3. Press (B), then select the "Health&Safety Inject" option
8. Select the FBI injectable `.app` for your region
4. Press (A) and confirm to inject
9. Press (Start) to reboot
10. If you still launch to the stock Health & Safety app and have downgraded with Gateway in the past, [follow this troubleshooting guide](troubleshooting#gw_fbi)

##### Section VI - Finalizing setup

2. Open Health and Safety (which is now FBI)
3. Select "SD"
4. Select "cias"
5. Navigate to `FBI.cia` and press (A) to install
6. Navigate to `hblauncher_loader.cia` and press (A) to install
7. Navigate to `lumaupdater.cia` and press (A) to install
8. Return to the SD directory with (B)
8. Navigate to `arm9loaderhax.bin`, then press (A) on it and select the "Copy" option
9. Return to the FBI main menu with (B)
10. Select "CTR NAND"
11. Select "\<current directory>"
12. Select the "Paste" option, then press (A) to confirm
8. Exit with the home button
9. Launch the Homebrew Launcher from the home menu
10. Select "DSP Dump"
11. Press (Start) when prompted to exit
12. Reboot while holding Start to launch Hourglass9
13. Go to "SysNAND Backup/Restore", then select "Health&Safety Inject"
14. Select `hs.app` (the original one that doesn't contain FBI), then press (A) and confirm to inject
15. Press (Select) on the main menu to eject your SD card
15. Press (Start) to reboot with the SD card removed
  + Booting the device at least once with the SD card removed will allow you to configure the CTRNAND based luma installation
16. Use the (A) button and the D-Pad to turn on the following:    
  + **"Show NAND or user string in System Settings"**
3. If you are using a **New 3DS**, you should *also* enable the following:
  + **"New 3DS CPU" to "Clock+L2(x)"**
    + This will increase the framerate of many games, but may cause instability in others
    + If some games do not work properly, disable this option and try again
14. Reinsert your SD card, then press Start to save and reboot!
